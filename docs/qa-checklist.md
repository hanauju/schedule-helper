# QA Checklist

## App Launch

- [ ] 앱이 오류 없이 실행된다.
- [ ] 기본 사용 가능 시간이 평일 09:00-17:00으로 생성된다.
- [ ] 샘플 데이터 버튼으로 작업과 고정 일정이 추가된다.

## Tasks

- [ ] 작업을 새로 추가할 수 있다.
- [ ] 선택한 작업을 수정할 수 있다.
- [ ] 선택한 작업을 완료 처리할 수 있다.
- [ ] 선택한 작업을 삭제할 수 있다.

## Events

- [ ] 빈 캘린더 칸을 클릭하면 새 일정 시간이 자동으로 채워진다.
- [ ] 고정 일정을 저장할 수 있다.
- [ ] 선택한 일정을 수정할 수 있다.
- [ ] 선택한 일정을 삭제할 수 있다.
- [ ] 겹치는 일정 저장 시 경고가 나온다.

## Auto Scheduling

- [ ] 자동 배치를 실행하면 미완료 작업이 주간 캘린더에 생성된다.
- [ ] 고정 일정과 자동 배치 일정이 겹치지 않는다.
- [ ] 마감일이 지난 작업은 미배치로 표시된다.
- [ ] 사용 가능 시간이 부족한 작업은 미배치로 표시된다.

## Persistence

- [ ] 앱을 재실행해도 작업이 유지된다.
- [ ] 앱을 재실행해도 일정이 유지된다.
- [ ] 앱을 재실행해도 사용 가능 시간 설정이 유지된다.
